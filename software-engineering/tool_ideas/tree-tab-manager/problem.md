# Problem Document - Tree Tab Manager

## Central problem
Managing tabs as a developer is very common an painful.

I have certain tabs for specific API reference, API general doc, API test account.
If they all belong to the same company, I only have the favicon of the company to identify them.
This creates a manual drawer opening experience to get to each site.

I want to be able to have a tree-structure similar to a file explorer. Each file is a tab which can be 
named and indexed. They can be searched and sorted etc. they can be traversed by nested groups and name.


## Current Best Solutions
Over bloated enterprise solutions that are not dev focused.
Other apps that dont exactly capture the tree structure experience.

# Observations
Unlike files, the URLs are not always the same. People can traverse to different pages.
To handle this, we can have a macro which allows you to jump back to to the primary URL which the tab 
was created from or assigned to.
