# Problem Document - Luasnip like plugin for vim that has a more structured and practical AI gen flow

## Central problem
The problem I have with Cursor like chat experiences is that you lose sight of your code and the context of your code.
I beleive in using code-gen in small units (units like unit test sized scopes). Think of units as function and module scopes.
This promotes a more healthy ratio of code-gen to code review and human in the loop.

You shouldn't be able to give a massively general prompt like "make me a whole" app or "auth system".

I mention luasnip because it would be a similar experience to using a snippet manager like luasnip, you can store your 
(prompts and context pulling directives) (prompt + context are the inputs to a LLM function) in a structured way, which can be recalled and re-used.
