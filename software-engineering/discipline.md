# Development Discipline

This is intended to capture the set of rules, guidelines, and principles to try and adhere to in order to ensure code quality.

# KPIs
## Correctness
Does the software work according to the **known** requirements?
## Readability
How fast can the software be understood by another developer? How much accompanied context does the code require for it to be understood?
## Flexibility
How much effort does it take to change the software in the face of **new** requirements?
## Stability
Does the software work in the face of **unknown** conditions to its internal or external state? How bug prone is it?
## Observability
How easy is it for analysts to understand what the software is doing in a live environment? Do the logs tell a story? 
## Security
How easy is it for an attacker to compromise the software?
