# Software Development Life-cycle

## Metrics for Increasing Project Observability
### Development Speed
- Requirements Rate of change (Change event per <sprint-cycle>)
Events that will cause context-switching within the development process. Anything that will cause developers to switch gears.
This is costly towards the development speed of ongoing tasks and should be used sparingly: 
    - Change in priority
    - Change in task requirements

- Production Rate of Change (Change events per <sprint-cycle>)
Events that actually change production as a result of development. Counting only production changes for this metric ensures that it expresses the performance of real product change. This gives stakeholders
a much more accurate sense of how fast development takes for the various requirements they put on the roadmap.
