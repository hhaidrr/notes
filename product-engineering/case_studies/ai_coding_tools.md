# Case Study - AI coding tools

## Pre-condition
AI coding tools are rapidly coming out. I am a classical coder and I haven't seen first hand what the possible
capabilities of these tools really are.

## Experiment
We are going to take one of our product ideas documents and run it through a list of the latest and most popular 
AI tools out there to see what they can do.
Tools to use:
- Lovable
- Bolt
- v0
- Replit
- Supabase

## Post-condition (success)
I am pretty blown away by what the tools could do with my product document and a few chat iterations.
I was able to instantly scaffold a landing page and some basic web app functionality, even 
a database schema, auth, page navigation. I can viably build a MVP much faster using these tools than I could 
in a classical manner. 

Some things to consider are if using multiple tools, syncing them to a single repository is not clear.
Just prompting broadly is fine when starting with a product spec document. Though after a bit, 
a ticket based iteration process like Jira or Gitlab Projects will be needed to fine-tune various bits
