# Service Locator

There is often not any tools to find the best path towards the optimal tool. 

We have many apps, many tools, many services, many abstractions. Often it is up to the client 
to the client to deal with finding their optimal set of algorithms based on their current state. 
This is okay and it would be difficult to match a client's problem state to a solution set, 
but would be very valuable. However the intermediary of that is to expose and make clear 
the pros and cons, optimal use case for existing apps, services etc.
