# Physical Activity Watcher
This document outlines the requirements for a system which is able to monitor the physical movements of a user
in order to infer what activities it is performing in the physical world.

## Terminology

## Success Criteria
- The automation is able to get from start to end of an application form with 0% applicant intervention 100% of the time for %100 percent of company sites.
- The automation is able to get from start to end of an application form with 90% >= success rate without a human service operator
on non-indexed company sites.
- The automation is able to get from start to end of an application form with 99% >= success rate without a human service operator
on indexed company sites.

## Observations/Backlog
- The system can have the user help it get better at inferring user activity by asking it at different times or showing 
it a real-time text of what it thinks the user is doing based. The user can then course-correct it by telling it what 
it is really doing. This comes into mind when user movements don't really tell much as to what they might be doing.
Such as reading at my desk. I may be reading, while just scratching my head and laying back. Or I may be doing
something completely different. Passive tasks like this that don't involve hands will be trickier to infer. So 
having the user help the system get better is an avenue to correct the system.
