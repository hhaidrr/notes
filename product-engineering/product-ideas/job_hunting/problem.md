# Problem Document - AI job application platform

## Central problem
Recurring broadcast applications to different positions online is often manual even with autofill features. It still needs supervision and manual intervention from the user.

The user should not have any exposure to the application process, they shouldnt even see the company job site.

Especially when today software engineers have to apply 1000 times before scraping by a few interviews. It is a grind to do this manually.

# Ideal solution

## Feature Backlog
as new jobs come up that the agent thinks are relevant, I can swipe left or right to add to the existing broadcast list
agent will look through inbox or site to see offer status and update table
agent will automatically take action upon rejection/acceptence
on rejection, sets a re-apply schedule on a default 6 month internval
on acceptance, will suggest times to set an interview date.
Agent will keep up with networking activities like keeping up with what current contacts are doing, achieving, and 
suggesting generated replies to keep in touch and build rapport.
