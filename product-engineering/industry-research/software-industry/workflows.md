# Workflows - Software 

Software (with underlying hardware) is a meta algorithm which allows us to write self 
executing algorithms that do not need human itervention or costs. Software costs come from
indirect dependencies which at some point still require human labour. However the software itself
does not require human costs.

Software is dynamic enough to deal with a monumental amount of different workflows for different
industries. It is the interpreter pattern which is so powerful.

Currently the way we write software is direct. It is slowly moving towards a generative approach
due to AI that can write code. However it is not there yet.

## What Software Can't do
There are a few things software cannot cover, which is more manageable to document than to 
discuss the things it can cover.

### 1. Iteract with Physical Reality
- Material Existence: Software cannot directly manipulate physical matter; 
it requires hardware or machinery as an intermediary.
- Example: Building a house requires manual labor and physical materials, though software 
can assist with design and planning (e.g., CAD tools).
- Biological Constraints: Software can't fully replicate or replace complex, organic processes like natural photosynthesis or biological growth.
<br>
### 2. Extremely Unpredictable Environments
- Chaotic Systems: In scenarios governed by extreme randomness or chaos 
(e.g., turbulent weather or stock market flash crashes), software may struggle to predict 
or model outcomes accurately.
- Unknown Phenomena: In cases where data is scarce or understanding is incomplete, software's 
ability to analyze or act is limited. For instance:
    - Studying unknown extraterrestrial phenomena.
    - Dealing with highly rare, unpredictable events ("black swans").
- e.g. Human emotion/intuition, Ethical and social understanding
<br>
### 3. When Cons Outweigh the Benefit
- In some cases, applying software might not be worth the investment:
- Automating extremely simple or infrequent tasks may be overkill.
- Over-reliance on software in situations where manual processes are faster, cheaper, 
or more practical.
