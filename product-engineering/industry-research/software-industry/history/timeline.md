# Timeline - Development of the Software Industry

## Software as a Product (for users)
Software itself does not necessarily have workflows associated to it when looking at its 
relationship to its **users**. Software can take many workflows for its users. So it is not tied
to a handful of workflows, it is much too dynamic for that.

## Software Development Services (for developers)
On the other hand software developers have a key handful of workflows which can be identified. 
Creating services for software developers is possible in this case.

There can be two separate timeline docs for these two different actors (users, developers). this
doc will focus on the former.
